API Reference / Party Lifecycle Management / Create party
POST /v1/parties/initiate

Create party

Create a draft party in the bank's Party Lifecycle Management service. Returns a canonical party object with a generated identifier and status draft. The party is not yet active and cannot transact until verification and screening are complete (see the onboarding journey).

Parameters

Headers
Authorization
stringRequired
Bearer token from your OAuth2 client-credentials exchange. Token must include the plm:write scope.
X-Tenant-Id
stringRequired
The tenant identifier issued at onboarding (e.g. bank_a). Glue routes the request to the configured core for that tenant.
Idempotency-Key
stringOptional
A unique key (e.g. UUID v4) to deduplicate retried requests. Honoured for 24 hours.
Body parameters
givenName
stringRequired
The party's given (first) name. 1–100 UTF-8 characters.
familyName
stringRequired
The party's family (last) name. 1–100 UTF-8 characters.
dateOfBirth
string <date>Required
ISO-8601 date (YYYY-MM-DD). Must indicate an age between 18 and 120 inclusive.
residentialAddress
objectRequired
Country (ISO-3166 alpha-2) is mandatory; city, postal code, and street are optional and may be enriched later.
contact
objectOptional
May include email and phone (E.164). At least one is required before lifting status to active.
externalRef
stringOptional
A free-form reference (max 64 chars) for correlating the party with your CRM record.

Request example

JSON
{
  "givenName": "Asha",
  "familyName": "Mehta",
  "dateOfBirth": "1992-04-12",
  "residentialAddress": {
    "country": "AE",
    "city": "Abu Dhabi",
    "postalCode": "00000"
  },
  "contact": {
    "email": "asha.mehta@example.com",
    "phone": "+971501234567"
  },
  "externalRef": "crm_7b3e"
}

Responses

JSON · 201
{
  "id": "cust_8a2f",
  "status": "draft",
  "givenName": "Asha",
  "familyName": "Mehta",
  "dateOfBirth": "1992-04-12",
  "createdAt": "2026-04-29T08:14:22Z",
  "_links": {
    "self": "/v1/parties/cust_8a2f",
    "lifecycle": "/v1/parties/cust_8a2f/lifecycle"
  }
}
JSON · 400
{
  "error": "validation_failed",
  "message": "One or more required fields are invalid.",
  "details": [
    {
      "field": "dateOfBirth",
      "reason": "must indicate an age of 18 or older"
    }
  ]
}
JSON · 409
{
  "error": "duplicate_party",
  "message": "A party with the same externalRef already exists for this tenant.",
  "existingPartyId": "cust_7b3e"
}
BIAN mapping This endpoint maps to the BIAN Party Lifecycle Management service domain, InitiateCustomerReference service operation. The canonical response shape preserves BIAN field names where they exist; tenant extensions appear under _extensions.{tenantId} and never collide with the canonical contract.
